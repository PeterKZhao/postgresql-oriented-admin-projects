# 需求模型卡模板

本文是 `requirements.txt` 逐项建模时使用的统一模板。每个需求点都应按此结构保存，方便后续 SQL 设计、Java 服务设计和测试设计。

## 需求名称

填写需求名称，例如：多语言化、信息发布、门票核销、年卡延期、座位预订。

## 需求来源

填写 `requirements.txt` 中的章节路径。

## 模型类型

可选值：

- `none`：无数学模型，普通 CRUD/配置/内容管理。
- `state`：业务状态模型，例如审核、发布、订单、售后。
- `formula`：公式计算模型，例如价格、退款、积分、佣金、滞纳金。
- `quota`：库存、容量、次数、权益模型。
- `route`：地图、距离、路线、最近点模型。
- `optimization`：排班、路径、资源分配等优化模型。

## 不变量

- 永远必须成立的业务约束。
- 金额不能超退。
- 次数不能为负。
- 座位不能重复售卖。
- 已删除/禁用/过期数据不能前台使用。

## 异常场景

- 输入缺失。
- 状态非法。
- 重复提交。
- 并发冲突。
- 外部系统失败。
- 数据被删除或下线。
